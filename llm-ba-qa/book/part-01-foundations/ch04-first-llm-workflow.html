4.1 Setting Up Your Environment

Before writing your first line of code, you need a clean, organized development environment. Even if you're not a developer by trade, setting up a proper environment is straightforward and will save you hours of frustration later.

What You Need:

  • Python 3.10 or later — The language we'll use for all code examples. It's the dominant language for AI/ML work and has the best library ecosystem for LLM integration.

Secure Key Management:

"""
Secure API key management using environment variables.
"""
import os
from dotenv import load_dotenv

# Load API keys from .env file
load_dotenv()

def get_api_key(provider):
    """Securely retrieve an API key with validation."""
    key_names = {
        "openai": "OPENAI_API_KEY",
        "anthropic": "ANTHROPIC_API_KEY",
        "google": "GOOGLE_API_KEY"
    }

    env_var = key_names.get(provider.lower())
    if not env_var:
        raise ValueError(f"Unknown provider: {provider}")

    key = os.getenv(env_var)
    if not key:
        raise EnvironmentError(
            f"{env_var} not found. Please add it to your .env file.\n"
            f"Example: {env_var}=sk-your-key-here"
        )

    # Basic validation (don't log the full key!)
    if len(key) < 20:
        raise ValueError(f"{env_var} appears to be invalid (too short)")

    # Log that we found a key without exposing it
    masked = key[:8] + "..." + key[-4:]
    print(f"Loaded {provider} API key: {masked}")

    return key

# Usage
openai_key = get_api_key("openai")

Cost Controls:

Before you start making API calls, set up spending controls to avoid surprises:

  • OpenAI: Set a monthly spending limit in Settings > Billing > Usage limits
  • Anthropic: Set usage limits in Console > Settings > Spending
  • In your code: Track token usage and set per-session budgets
"""
Cost tracking wrapper for LLM API calls.
"""

class CostTracker:
    """Track API usage costs across a session."""

    # Approximate costs per 1M tokens (update as pricing changes)
    PRICING = {
        "gpt-4o":          {"input": 2.50,  "output": 10.00},
        "gpt-4o-mini":     {"input": 0.15,  "output": 0.60},
        "claude-sonnet":   {"input": 3.00,  "output": 15.00},
        "claude-haiku":    {"input": 0.25,  "output": 1.25},
    }

    def __init__(self, budget_usd=1.00):
        self.budget = budget_usd
        self.total_cost = 0.0
        self.calls = []

    def record_call(self, model, input_tokens, output_tokens):
        """Record an API call and its cost."""
        rates = self.PRICING.get(model, {"input": 5.0, "output": 15.0})
        cost = (
            (input_tokens / 1_000_000) * rates["input"] +
            (output_tokens / 1_000_000) * rates["output"]
        )
        self.total_cost += cost
        self.calls.append({
            "model": model,
            "input_tokens": input_tokens,
            "output_tokens": output_tokens,
            "cost": cost
        })

        if self.total_cost > self.budget:
            print(f"WARNING: Session budget of ${self.budget:.2f} exceeded! "
                  f"Total: ${self.total_cost:.4f}")

        return cost

    def summary(self):
        """Print a cost summary for the session."""
        print(f"\nSession Cost Summary")
        print(f"{'='*40}")
        print(f"Total API calls: {len(self.calls)}")
        print(f"Total cost: ${self.total_cost:.4f}")
        print(f"Budget remaining: ${self.budget - self.total_cost:.4f}")
        total_in = sum(c["input_tokens"] for c in self.calls)
        total_out = sum(c["output_tokens"] for c in self.calls)
        print(f"Total tokens: {total_in:,} in / {total_out:,} out")

4.3 Making Your First API Call

With your environment set up and API keys secured, let's make your first programmatic LLM call. We'll start with the simplest possible example, then build complexity.

"""
Your first LLM API call — analyzing a requirement.
"""
from openai import OpenAI
from dotenv import load_dotenv

load_dotenv()
client = OpenAI()  # Automatically uses OPENAI_API_KEY from environment

# A simple requirement to analyze
requirement = """
The system shall allow users to reset their password by clicking
a "Forgot Password" link on the login page.
"""

# Make the API call
response = client.chat.completions.create(
    model="gpt-4o-mini",           # Cost-effective for learning
    temperature=0.2,                # Low temperature for analytical tasks
    messages=[
        {
            "role": "system",
            "content": "You are a Senior Business Analyst. Analyze requirements "
                       "for completeness, ambiguity, and testability. Be specific "
                       "and constructive in your feedback."
        },
        {
            "role": "user",
            "content": f"Analyze this requirement:\n\n{requirement}"
        }
    ]
)

# Extract the response
analysis = response.choices[0].message.content
print("Analysis:")
print(analysis)

# Check usage
print(f"\nTokens used: {response.usage.prompt_tokens} input, "
      f"{response.usage.completion_tokens} output")
print(f"Total: {response.usage.total_tokens} tokens")

Key differences between the APIs:

Feature OpenAI Anthropic
System message Included in messages array with role "system" Separate system parameter
Max tokens Optional (model has defaults) Required — must specify max_tokens
Response access response.choices[0].message.content response.content[0].text
Token tracking response.usage.prompt_tokens response.usage.input_tokens
JSON mode response_format={"type": "json_object"} Use prompt instructions (or tool_use for structured output)

4.4 Building a Simple Analyzer

Now let's build something genuinely useful: a tool that reads a list of requirements and produces a quality assessment for each one. This is a real workflow you can use immediately in your daily work.

"""
Requirements Quality Analyzer
Reads requirements from a text file and produces a structured quality report.
"""
import json
from openai import OpenAI
from dotenv import load_dotenv

load_dotenv()
client = OpenAI()

ANALYSIS_PROMPT = """You are a Senior Business Analyst performing a requirements
quality review. Analyze the following requirement and return a JSON object with
this exact structure:

{
  "requirement_id": "the ID if provided, otherwise 'UNID'",
  "original_text": "the requirement as provided",
  "quality_scores": {
    "clarity": 1-5,
    "completeness": 1-5,
    "testability": 1-5,
    "consistency": 1-5,
    "feasibility": 1-5
  },
  "overall_score": 1-5,
  "issues": [
    {
      "type": "ambiguity|incompleteness|untestable|vague|conflicting",
      "description": "specific description of the issue",
      "suggestion": "how to fix it"
    }
  ],
  "improved_version": "a rewritten version that addresses all issues",
  "verdict": "pass|needs_revision|fail"
}

Scoring guide:
5 = Excellent: no issues found
4 = Good: minor improvements possible
3 = Acceptable: some issues that should be addressed
2 = Poor: significant issues that must be addressed
1 = Failing: requirement is not usable as written

Requirement to analyze:"""

def analyze_requirement(requirement_text, model="gpt-4o-mini"):
    """Analyze a single requirement and return structured results."""
    response = client.chat.completions.create(
        model=model,
        temperature=0,
        response_format={"type": "json_object"},
        messages=[
            {"role": "system", "content": ANALYSIS_PROMPT},
            {"role": "user", "content": requirement_text}
        ]
    )

    result = json.loads(response.choices[0].message.content)
    result["_meta"] = {
        "model": model,
        "input_tokens": response.usage.prompt_tokens,
        "output_tokens": response.usage.completion_tokens
    }
    return result

def analyze_requirements_file(filepath, model="gpt-4o-mini"):
    """Analyze all requirements in a file (one per line or separated by blank lines)."""
    with open(filepath, "r") as f:
        content = f.read()

    # Split on blank lines to handle multi-line requirements
    requirements = [r.strip() for r in content.split("\n\n") if r.strip()]

    results = []
    for i, req in enumerate(requirements, 1):
        print(f"Analyzing requirement {i}/{len(requirements)}...")
        result = analyze_requirement(req, model)
        results.append(result)

    return results

# Example: Analyze sample requirements
sample_requirements = [
    "REQ-001: The system shall be fast and user-friendly.",

    "REQ-002: The system shall return search results within 200ms "
    "for queries matching a single field, and within 500ms for "
    "full-text search across all indexed fields, measured at the "
    "95th percentile under a load of 500 concurrent users.",

    "REQ-003: Users should be able to do stuff with their accounts.",

    "REQ-004: The payment processing module shall support Visa, "
    "Mastercard, and American Express credit card transactions. "
    "Failed transactions shall display error code and a user-friendly "
    "message. All transaction data shall be encrypted using AES-256 "
    "and stored in compliance with PCI DSS Level 1 standards.",
]

# Run analysis
all_results = []
for req in sample_requirements:
    result = analyze_requirement(req)
    all_results.append(result)
    score = result["overall_score"]
    verdict = result["verdict"]
    print(f"  {result['requirement_id']}: Score {score}/5 — {verdict}")
Iteration in Practice: Your first version of an analyzer won't be perfect. Run it on 10-20 real requirements from your project, review the outputs carefully, and adjust the prompt based on where the model's assessments differ from your expert judgment. This iterative refinement is normal and expected — it's how prompt engineering works.

4.5 Handling Responses and Errors

Production-quality workflows need robust error handling. LLM API calls can fail for many reasons: network issues, rate limits, invalid inputs, or unexpected response formats. Here's how to build resilience into your workflows.

"""
Robust LLM client with retry logic and error handling.
"""
import json
import time
from openai import OpenAI, APIError, RateLimitError, APIConnectionError
from dotenv import load_dotenv

load_dotenv()

class RobustLLMClient:
    """An LLM client with automatic retry, error handling, and cost tracking."""

    def __init__(self, max_retries=3, retry_delay=2.0):
        self.client = OpenAI()
        self.max_retries = max_retries
        self.retry_delay = retry_delay
        self.total_input_tokens = 0
        self.total_output_tokens = 0
        self.call_count = 0

    def call(self, messages, model="gpt-4o-mini", temperature=0.2,
             json_mode=False, max_tokens=2000):
        """Make an API call with automatic retry on transient errors."""

        kwargs = {
            "model": model,
            "temperature": temperature,
            "max_tokens": max_tokens,
            "messages": messages
        }
        if json_mode:
            kwargs["response_format"] = {"type": "json_object"}

        for attempt in range(1, self.max_retries + 1):
            try:
                response = self.client.chat.completions.create(**kwargs)

                # Track usage
                self.total_input_tokens += response.usage.prompt_tokens
                self.total_output_tokens += response.usage.completion_tokens
                self.call_count += 1

                content = response.choices[0].message.content

                # Parse JSON if expected
                if json_mode:
                    try:
                        return json.loads(content)
                    except json.JSONDecodeError as e:
                        print(f"Warning: JSON parse failed: {e}")
                        print(f"Raw response: {content[:200]}...")
                        if attempt < self.max_retries:
                            continue
                        return {"error": "JSON parse failed", "raw": content}

                return content

            except RateLimitError:
                wait = self.retry_delay * (2 ** (attempt - 1))  # Exponential backoff
                print(f"Rate limited. Waiting {wait}s before retry {attempt}...")
                time.sleep(wait)

            except APIConnectionError:
                wait = self.retry_delay * attempt
                print(f"Connection error. Waiting {wait}s before retry {attempt}...")
                time.sleep(wait)

            except APIError as e:
                print(f"API error (attempt {attempt}): {e}")
                if attempt == self.max_retries:
                    raise
                time.sleep(self.retry_delay)

        raise Exception(f"Failed after {self.max_retries} retries")

    def usage_summary(self):
        """Return a summary of API usage."""
        return {
            "total_calls": self.call_count,
            "total_input_tokens": self.total_input_tokens,
            "total_output_tokens": self.total_output_tokens,
            "total_tokens": self.total_input_tokens + self.total_output_tokens
        }

# Usage example
llm = RobustLLMClient(max_retries=3)

result = llm.call(
    messages=[
        {"role": "system", "content": "You are a helpful analyst assistant."},
        {"role": "user", "content": "List 3 common causes of ambiguous requirements."}
    ],
    model="gpt-4o-mini",
    temperature=0.3
)

print(result)
print(f"\nUsage: {llm.usage_summary()}")

Common Error Patterns and Solutions:

Error Cause Solution
RateLimitError Too many requests per minute Implement exponential backoff; add delays between batch calls
InvalidRequestError (context length) Input exceeds model's context window Truncate or chunk the input; use a model with a larger context window
AuthenticationError Invalid or expired API key Verify the key in your .env file; regenerate if needed
JSON parse failure Model returned text instead of valid JSON Use JSON mode; add "Return ONLY valid JSON" to the prompt; retry
Empty or truncated response max_tokens too low Increase max_tokens; check finish_reason for "length"
Check finish_reason: Always check response.choices[0].finish_reason. If it's "length" instead of "stop", the model's response was cut off because it hit the max_tokens limit. This means your output is incomplete — increase max_tokens or ask for a more concise response.

4.6 From Script to Reusable Tool

A script that runs once is useful. A tool that your whole team can use, that handles edge cases gracefully, and that produces consistent output is transformative. Let's refactor our requirements analyzer into a proper reusable tool.

"""
requirements_analyzer.py — A production-ready requirements quality analyzer.

Usage:
    python requirements_analyzer.py input.txt --output report.json --model gpt-4o-mini
    python requirements_analyzer.py input.txt --format html --threshold 3
"""
import argparse
import json
import sys
from datetime import datetime
from pathlib import Path

# (Assume RobustLLMClient from Section 4.5 is imported)

class RequirementsAnalyzer:
    """Analyze a set of requirements for quality and produce structured reports."""

    SYSTEM_PROMPT = """You are a Senior Business Analyst performing a formal
requirements quality review. You are thorough, constructive, and specific.

For each requirement, evaluate against these criteria:
- Clarity: Is the language unambiguous?
- Completeness: Are all necessary details specified?
- Testability: Can this be verified with a concrete test?
- Consistency: Does it avoid contradictions?
- Feasibility: Is it technically achievable?

Return your analysis as a JSON object with this structure:
{
  "requirement_id": "string",
  "original_text": "string",
  "quality_scores": {
    "clarity": 1-5,
    "completeness": 1-5,
    "testability": 1-5,
    "consistency": 1-5,
    "feasibility": 1-5
  },
  "overall_score": 1-5,
  "issues": [{"type": "string", "description": "string", "suggestion": "string"}],
  "improved_version": "string",
  "verdict": "pass|needs_revision|fail"
}"""

    def __init__(self, model="gpt-4o-mini", pass_threshold=3):
        self.llm = RobustLLMClient()
        self.model = model
        self.pass_threshold = pass_threshold

    def analyze(self, requirements):
        """Analyze a list of requirement strings."""
        results = []
        for i, req in enumerate(requirements, 1):
            print(f"  [{i}/{len(requirements)}] Analyzing: {req[:60]}...")
            result = self.llm.call(
                messages=[
                    {"role": "system", "content": self.SYSTEM_PROMPT},
                    {"role": "user", "content": req}
                ],
                model=self.model,
                temperature=0,
                json_mode=True
            )
            results.append(result)
        return results

    def generate_report(self, results):
        """Generate a summary report from analysis results."""
        total = len(results)
        passing = sum(1 for r in results if r.get("overall_score", 0) >= self.pass_threshold)
        failing = total - passing

        avg_scores = {}
        for dimension in ["clarity", "completeness", "testability", "consistency", "feasibility"]:
            scores = [r.get("quality_scores", {}).get(dimension, 0) for r in results]
            avg_scores[dimension] = round(sum(scores) / len(scores), 1) if scores else 0

        # Identify most common issue types
        all_issues = []
        for r in results:
            all_issues.extend([i.get("type", "unknown") for i in r.get("issues", [])])

        issue_counts = {}
        for issue in all_issues:
            issue_counts[issue] = issue_counts.get(issue, 0) + 1

        report = {
            "report_date": datetime.now().isoformat(),
            "model_used": self.model,
            "total_requirements": total,
            "passing": passing,
            "failing": failing,
            "pass_rate": f"{(passing/total)*100:.0f}%" if total > 0 else "N/A",
            "average_scores": avg_scores,
            "weakest_dimension": min(avg_scores, key=avg_scores.get) if avg_scores else None,
            "common_issues": dict(sorted(issue_counts.items(), key=lambda x: -x[1])[:5]),
            "api_usage": self.llm.usage_summary(),
            "details": results
        }
        return report

def parse_requirements_file(filepath):
    """Parse a requirements file — one requirement per paragraph."""
    text = Path(filepath).read_text()
    reqs = [r.strip() for r in text.split("\n\n") if r.strip()]
    return reqs

def main():
    parser = argparse.ArgumentParser(description="Analyze requirements quality")
    parser.add_argument("input", help="Path to requirements file")
    parser.add_argument("--output", "-o", default="report.json",
                        help="Output file path")
    parser.add_argument("--model", "-m", default="gpt-4o-mini",
                        help="LLM model to use")
    parser.add_argument("--threshold", "-t", type=int, default=3,
                        help="Minimum score to pass (1-5)")
    args = parser.parse_args()

    print(f"Requirements Quality Analyzer")
    print(f"{'='*40}")
    print(f"Input: {args.input}")
    print(f"Model: {args.model}")
    print(f"Threshold: {args.threshold}/5\n")

    requirements = parse_requirements_file(args.input)
    print(f"Found {len(requirements)} requirements to analyze.\n")

    analyzer = RequirementsAnalyzer(model=args.model, pass_threshold=args.threshold)
    results = analyzer.analyze(requirements)
    report = analyzer.generate_report(results)

    # Save report
    with open(args.output, "w") as f:
        json.dump(report, f, indent=2)

    # Print summary
    print(f"\n{'='*40}")
    print(f"RESULTS SUMMARY")
    print(f"{'='*40}")
    print(f"Pass rate: {report['pass_rate']} ({report['passing']}/{report['total_requirements']})")
    print(f"Weakest area: {report['weakest_dimension']}")
    print(f"Average scores: {report['average_scores']}")
    print(f"Most common issues: {report['common_issues']}")
    print(f"\nFull report saved to: {args.output}")

if __name__ == "__main__":
    main()

4.7 Measuring Quality and Cost

How do you know if your LLM workflow is actually good? You need metrics — and not just the kind the model reports. You need to measure the quality of the model's output against your expert judgment and the cost-effectiveness of the workflow compared to manual alternatives.

Quality Metrics for Analyst Workflows:

Metric What It Measures How to Calculate Target
Acceptance Rate % of LLM output used without modification Count of accepted items / total items generated >60% for first drafts
Edit Distance How much the analyst changes the LLM output Character/word changes between LLM output and final version <30% modification
Time Savings Actual hours saved vs. manual approach Time(manual) - Time(LLM-assisted) for equivalent quality output >50% reduction
Defect Detection Rate Issues found by LLM that humans missed (and vice versa) Compare LLM findings with manual review findings LLM catches 80%+ of human-found issues
False Positive Rate Issues flagged by LLM that aren't actually issues Count of false positives / total issues flagged <20% 
"""
Quality and cost measurement framework for LLM workflows.
"""
import time
from dataclasses import dataclass, field
from typing import List

@dataclass
class WorkflowMetrics:
    """Track quality and cost metrics for an LLM-assisted workflow."""
    workflow_name: str
    start_time: float = field(default_factory=time.time)
    items_generated: int = 0
    items_accepted_as_is: int = 0
    items_modified: int = 0
    items_rejected: int = 0
    issues_found_by_llm: int = 0
    false_positives: int = 0
    total_input_tokens: int = 0
    total_output_tokens: int = 0
    total_api_cost: float = 0.0
    manual_time_estimate_hours: float = 0.0

    @property
    def acceptance_rate(self):
        if self.items_generated == 0:
            return 0
        return self.items_accepted_as_is / self.items_generated

    @property
    def elapsed_hours(self):
        return (time.time() - self.start_time) / 3600

    @property
    def time_savings_pct(self):
        if self.manual_time_estimate_hours == 0:
            return 0
        return (1 - self.elapsed_hours / self.manual_time_estimate_hours) * 100

    @property
    def false_positive_rate(self):
        total_flagged = self.issues_found_by_llm + self.false_positives
        if total_flagged == 0:
            return 0
        return self.false_positives / total_flagged

    @property
    def cost_per_item(self):
        if self.items_generated == 0:
            return 0
        return self.total_api_cost / self.items_generated

    def report(self):
        """Generate a metrics report."""
        print(f"\n{'='*50}")
        print(f"WORKFLOW METRICS: {self.workflow_name}")
        print(f"{'='*50}")
        print(f"Items generated:      {self.items_generated}")
        print(f"Acceptance rate:      {self.acceptance_rate:.0%}")
        print(f"False positive rate:  {self.false_positive_rate:.0%}")
        print(f"Time elapsed:         {self.elapsed_hours:.2f} hours")
        print(f"Manual estimate:      {self.manual_time_estimate_hours:.2f} hours")
        print(f"Time savings:         {self.time_savings_pct:.0f}%")
        print(f"API cost:             ${self.total_api_cost:.4f}")
        print(f"Cost per item:        ${self.cost_per_item:.4f}")

        # ROI calculation (assuming $75/hour analyst rate)
        analyst_hourly_rate = 75.00
        manual_cost = self.manual_time_estimate_hours * analyst_hourly_rate
        llm_cost = self.elapsed_hours * analyst_hourly_rate + self.total_api_cost
        savings = manual_cost - llm_cost
        roi = (savings / llm_cost * 100) if llm_cost > 0 else 0

        print(f"\nROI Analysis (at ${analyst_hourly_rate}/hr):")
        print(f"  Manual cost:        ${manual_cost:.2f}")
        print(f"  LLM-assisted cost:  ${llm_cost:.2f}")
        print(f"  Savings:            ${savings:.2f}")
        print(f"  ROI:                {roi:.0f}%")

Project: Requirements Analyzer

Your capstone project for Part 1 brings together everything you've learned. Build a complete Requirements Analyzer tool that you can use on your real projects.

Requirements:

  1. Reads requirements from a text file (one requirement per paragraph)
  2. Analyzes each requirement for clarity, completeness, testability, consistency, and feasibility
  3. Produces both a JSON report and a human-readable summary
  4. Includes cost tracking and quality metrics
  5. Handles errors gracefully with retry logic

Extension Challenges:

  • Cross-reference check: After individual analysis, run a second pass that checks for inconsistencies between requirements (conflicting constraints, overlapping scope, missing dependencies)
  • HTML report: Generate an HTML report with color-coded quality scores (green/yellow/red) that you can share with stakeholders
  • Multi-model comparison: Run the same requirements through two different models and compare their assessments to identify where models disagree (these disagreements often highlight genuinely ambiguous requirements)
  • Historical tracking: Store results over time so you can track whether your team's requirements quality is improving across sprints

Summary
